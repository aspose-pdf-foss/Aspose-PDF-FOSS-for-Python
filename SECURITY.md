# Security Policy

## Supported Versions

Aspose.PDF FOSS for Python is currently in alpha. Security fixes are applied
to the default branch and included in the next published release.

| Version | Supported |
| --- | --- |
| Default branch | Yes |
| Latest PyPI release | Yes |
| Older releases | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.

Use
[GitHub private vulnerability reporting](https://github.com/aspose-pdf-foss/Aspose-PDF-FOSS-for-Python/security/advisories/new)
to send the maintainers:

- A description of the vulnerability and its potential impact.
- The affected version, commit, operating system, and Python version.
- A minimal reproducer or proof of concept when it is safe to share.
- Any known mitigations or suggested fixes.

Do not include confidential documents or third-party data in a report. Use a
minimal synthetic PDF or other fixture whenever possible.

The maintainers will acknowledge the report within five business days and
will coordinate validation, remediation, and disclosure with the reporter.
Please allow time for a fix to be prepared and released before publishing
details.

## Scope

Reports concerning malformed or hostile PDF input, parsing, rendering,
encryption, digital signatures, or package supply-chain behavior are
especially welcome.

Unsupported features and documented limitations are not vulnerabilities by
themselves. See
[supported-features.md](supported-features.md) for the current implementation
boundaries.
